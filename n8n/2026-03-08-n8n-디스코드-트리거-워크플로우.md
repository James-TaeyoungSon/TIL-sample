### Context
필자는 n8n을 사용하여 처음 워크플로우를 생성했습니다. 디스코드에서 메시지를 수신하면 트리거가 발생하고, 이를 통해 에이전트가 실행됩니다. 이 에이전트는 OpenAI API를 사용하여 공부한 내용을 정제하고 인사이트를 도출하여 README.md에 추가하는 기능을 합니다.

### Core
1. **워크플로우 생성**: n8n에서 'Add workflow' 버튼을 클릭하여 새로운 워크플로우를 생성합니다. 시작 지점으로 'Discord Trigger'를 추가하여 메시지 수신 시 트리거되도록 설정합니다.
2. **OpenAI API 연동**: 두 번째 단계로, OpenAI 노드를 추가하여 수신된 메시지를 정제하고 분석합니다. 이는 API 키를 통해 OpenAI와 통신하여 이루어집니다.
3. **결과 저장**: 마무리로, 분석된 결과를 README.md에 추가하는 파일 조작 노드를 사용하여 저장합니다.

### Insight
n8n은 처음 설정할 때 다소 복잡하게 느껴질 수 있습니다. 그러나 프로세스나 기능이 매우 직관적이라 사용하기 시작하면 강력한 도구임을 알 수 있습니다. 이러한 이유로 n8n의 활용도가 앞으로 더욱 높아질 것으로 기대합니다. 이는 자동화의 이점인 시간 절약, 비용 절감, 오류 최소화 및 운영 확장성에 기인합니다.
